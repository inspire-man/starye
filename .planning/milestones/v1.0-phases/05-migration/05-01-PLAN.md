<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions -> Cloudflare deploy/rollback APIs | CI/CD 拥有生产更改能力，必须保持输入清晰、行为可预期 |
| Operator -> rollback workflow | 错误 app 或错误 version_id 会直接影响生产恢复 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | rollback workflow inputs | mitigate | `app` / `version_id` 作为显式 workflow_dispatch 输入，回滚对象必须明确记录在日志中 |
| T-05-02 | Denial of Service | inconsistent wrangler versions | mitigate | 统一 wrangler floor，避免 deploy/rollback 行为因 CLI 能力差异漂移 |
| T-05-03 | Repudiation | manual rollback ambiguity | mitigate | 新增统一 rollback 入口；无法自动化的 Pages 路径也必须在 workflow 中显式说明 |

</threat_model>
