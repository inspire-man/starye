<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser player events → progress API | 播放进度是高频用户侧事件，需要防止错误路径或错误源污染历史 |
| movie history UI → auth-scoped API data | 继续观看 / 历史只能显示当前用户自己的记录 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-05 | Information Disclosure | movie history / continue rail | mitigate | 继续使用登录态 API，前端不缓存或拼接他人历史，history UI 只消费当前用户 route response |
| T-04-06 | Tampering | `streamUrl` progress pollution | mitigate | `streamUrl` 只在既有可信来源前提下纳入统一进度；progress scope 仍以 `movieCode` 为唯一影片标识 |
| T-04-07 | Repudiation | completed 状态被本地猜测导致错判 | mitigate | Home / History / Profile 全部改读显式 `completed`，去除 3600 秒猜测路径 |
| T-04-08 | Denial of Service | 高频 `timeupdate` 导致过度写库 | mitigate | 采用 10 秒 checkpoint + pause/seek/pagehide flush，不在每次 `timeupdate` 都直接写库 |

</threat_model>
