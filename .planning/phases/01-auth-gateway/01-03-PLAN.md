    ```ts
    /**
     * Phase 1 / AUTH-02 / D-01..D-04
     *
     * SSR 阶段预取 /api/auth/get-session：
     * - D-01: 把浏览器 cookie 原样 forward 到 gateway（只取 cookie 头）
     * - D-02: baseURL 走 runtimeConfig.public.apiUrl（gateway 原点），不直连 api
     * - D-03: 3s 超时 / 5xx / 网络错 → 降级匿名，不抛异常
     * - D-04: event.context 单例，请求内只调用一次
     *
     * 安全：仅 pick cookie 头，防止 SSR 转发其他敏感头到上游（T3 缓解）。
     */

    import { defineEventHandler, getHeader, getRequestURL } from 'h3'

    interface SessionPayload {
      user: unknown
      session: unknown
    }

    export default defineEventHandler(async (event) => {
      const url = getRequestURL(event)
      if (url.pathname.startsWith('/api') || url.pathname.includes('.')) {
        return
      }

      const cookie = getHeader(event, 'cookie')
      if (!cookie) {
        event.context.session = null
        return
      }

      const config = useRuntimeConfig()
      const apiUrl = config.public.apiUrl as string

      try {
        const session = await $fetch<SessionPayload | null>(
          '/api/auth/get-session',
          {
            baseURL: apiUrl,
            headers: { cookie },
            signal: AbortSignal.timeout(3000),
            retry: false,
          },
        )
        event.context.session = session ?? null
      }
      catch {
        event.context.session = null
      }
    })
    ```

    **文件 B：`apps/blog/app/plugins/session.ts`**（新建）

    ```ts
    /**
     * Phase 1 / AUTH-02 / D-04
     *
     * 把 SSR 阶段预取的 session 同步到 useState，供组件通过 useState('session') 直接读。
     * 客户端 hydrate 后 Better Auth 的 authClient.useSession() 会自行刷新，此 plugin 只消除首帧闪烁。

      name: 'session-seed',
      enforce: 'pre',
      setup() {
        const session = useState<unknown>('session', () => null)
        if (import.meta.server) {
          const event = useRequestEvent()
          session.value = event?.context.session ?? null
        }
      },
    })
    ```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| 浏览器 → Nuxt SSR | untrusted cookie 到达 Nitro event |
| Nuxt SSR → Gateway | middleware 通过 $fetch 跨 Worker 调用 |
| Nuxt Client ← Nuxt SSR | useState seed 跨 SSR→CSR 传递 session |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-09 (T3) | Information Disclosure | `server/middleware/session.ts` 的 forward header 构造 | mitigate | `headers: { cookie }` 仅 pick cookie，严禁 spread `getRequestHeaders(event)`；acceptance_criteria 通过 `grep -c "headers: { cookie }"` 强制验证 |
| T-01-10 (T3 变种) | Information Disclosure | `baseURL` 目标 | mitigate | 固定走 `useRuntimeConfig().public.apiUrl`（gateway 原点），不允许 `api.starye.org` 直连 —— grep 禁用 process.env |
| T-01-11 | Denial of Service | SSR fetch 长时间阻塞页面 | mitigate | D-03 `AbortSignal.timeout(3000)` + try/catch 降级匿名，SSR 响应仍 200 |
| T-01-12 | Tampering | hydrate 阶段 useState 被覆盖 | accept | `useState('session')` 仅 server 分支赋值；client 端 Better Auth 自行刷新。即使被篡改也只影响 UI 态（授权权威在 cookie） |
