## Artifacts this plan produces

- API/Gateway runtime adoption that reads selected-profile values emitted by the generated deploy configuration, preserving existing CORS credentials, auth forwarded-host behavior, proxy headers, and local gateway routing.
- Vite/Nuxt consumer adapters that use Plan 12-01's one closed `AuditedPublicRuntimeInput` injection path instead of fallback chains or ad hoc `import.meta.env` reads. Their build-time names are deliberately identical to the selected-surface generated dotenv protocol: each surface gets the four target fields plus only its named audited Sentry/release/build-mode fields; movie additionally gets its named monitoring and five feature fields. No adapter reads a legacy raw public value or accepts an arbitrary public map.
- A negative fixture assertion that the reconciled Pages project identities, including Blog `blog-pages`, never enter `TargetPublicRuntimeConfig`, Vite values, Nuxt `runtimeConfig.public`, request-base construction, or a browser-visible target selector (D-01/D-04/D-05).
- A complete, fixture-backed public-env inventory and static guard: auth/blog Nuxt config and clients; dashboard Vite config, bootstrap, router, Actors and Publishers; movie Vite config, bootstrap, router, Player, monitoring, Aria2/rating/performance/feature-flag consumers; and comic Vite config, bootstrap and router. Only the named framework adapters may read raw public env; all other browser consumers use typed adapter output.
- Exact workspace dependency declarations for auth, blog, dashboard, movie and comic (`@starye/config: workspace:*`) plus the corresponding five `pnpm-lock.yaml` importers.
- Fixture tests that switch between two target identities without credentials or source edits, and static/build/type checks proving public config cannot leak forbidden fields.
