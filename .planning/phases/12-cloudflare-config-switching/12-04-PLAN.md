<registry_contract>
TargetRemoteEntry is a closed union from a readonly TargetRemoteEntryDefinition registry in mutation-entry.ts. Each definition holds id, family, prepared child module, fixed child operation, execution mode (mutation, read-only, diagnostic), exact allowed option schema, fixed secret-key names, and a TargetedPreflightPolicy. The policy is data, not caller argv: it declares selected-profile resource checks and required fixed-secret presence/account checks. Callers cannot lower it, skip checks, switch a resource, or add child argv.

Registry entries are: d1-migrate, d1-migrate-movies-metadata, d1-cleanup-backup-preview, d1-cleanup-backup-execute, d1-cleanup-invalid-covers, crawler-comic, crawler-optimized, crawler-actor, crawler-publisher, crawler-search-index, crawler-enrich-players, crawler-check-config, crawler-backfill-covers, crawler-backfill-publishers, crawler-enrich-players-javbus, crawler-r2-storage-audit, and monthly-cleanup.

crawler-check-config is diagnostic: its policy validates selected crawler/API/R2 profile requirements, declared secret-key presence, ambient-identity rejection, and injected read-only resource checks. The child consumes only typed preparation/status data and emits key names plus boolean presence/equality state. It must not import dotenv or crawler code, or call fetch, browser, ImageProcessor, MappingFileManager, S3, D1, or an API. Missing required values remain fail-closed preflight issues, never secret reads by the child.
</registry_contract>

<source_audit_contract>
crawler-source-entry-contract.test.ts creates a TypeScript `ts.Program` over every tracked TypeScript source under packages/crawler, including scripts, src, and examples, excluding node_modules, dist, coverage, generated declarations, and Vitest test modules. It obtains a `TypeChecker`, resolves `import`/export-barrel aliases with `checker.getAliasedSymbol()`, resolves called symbols to declarations, and traces each direct executable's relevant local import graph. It must not use manifest aliases, text matching, or an untyped AST walk as the classification authority.

A direct executable has a top-level launch expression, import.meta/process.argv dispatch, or shebang-style CLI entry. A selected-target sink is resolved dotenv/config; resolved reads of API_URL, CRAWLER_SECRET, SERVICE_TOKEN, CLOUDFLARE_*, R2_*, DATABASE_*, or STARYE_TARGET_ID; a resolved ImageProcessor, MappingFileManager, S3Client, or crawler API client constructor/call; or resolved fetch/got flow to selected Starye API, Cloudflare D1, or R2. The classifier must distinguish those remote sinks from harmless worktree-only utilities, Vitest utilities, and external-source crawling that passes a local/external guard before any selected identity is read.

Each discovered executable has an explicit table classification:
- registry-child: target-crawl-mutation.ts alone parses argv, requiring exact prepared context and registry-owned operation before import.
- prepared-remote-operation: exported operation accepting PreparedCrawlerRuntime and closed options; no dotenv side effect or top-level launch.
- blocked-import-only: direct execution exits non-zero before dotenv, target construction, network/R2/D1/browser setup.
- local-or-external-only: local guard rejects remote identities/secrets before argv/environment work, permits validated worktree paths or external-source URLs, and has no selected Starye sink.
- library-or-test-module: no direct execution.

The current table must name src/index.ts, run-optimized.ts, run-actor.ts, run-publisher.ts, build-search.ts, src/scripts/enrich-players.ts, src/scripts/enrich-players-javbus.ts, backfill-covers.ts, backfill-publishers.ts, audit-r2-storage.ts, configure-user-permissions.ts, test-api.ts, test-api-auth.ts, test-full-flow.ts, test-image-upload.ts, test-movie-sync.ts, test-r2-mapping-storage.ts, test-real-movie-sync.ts, test-single-movie.ts, verify-data-integrity.ts, and verify-r2-upload.ts. Discovery defaults to fail for any future unclassified candidate, unresolved imported sink, or import graph that reaches an ambient selected-target identity.

Fixture programs must prove detection when the crawler API client or `ImageProcessor` is imported under an alias (including a re-export barrel), when dotenv/config or an ambient identity is reached indirectly, and when a prepared operation adds a top-level launch. They must also prove a guarded local/test/external helper is not falsely classified as remote. A direct executable may pass only as registry-child, prepared-remote-operation, blocked-import-only, or proven local-or-external-only; no fifth implicit allow path exists.
</source_audit_contract>
