<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI args -> preflight | Operator input selects target, scope, command, local profile, and CI environment. |
| Local shell env -> identity validation | Env vars may shadow Wrangler profile behavior or provide CI tokens. |
| CLI -> Wrangler live checks | Preflight invokes Cloudflare read checks through a command executor. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-11-07 | Spoofing | `preflight.ts` local identity | high | mitigate | Local mode rejects token/profile mixing and requires expected Wrangler profile per D-13/D-14. |
| T-11-08 | Tampering | remote command target | critical | mitigate | Remote high-risk commands require explicit target, CI environment mapping, credentials, and live resource checks per D-12/D-15/D-16. |
| T-11-09 | Repudiation | CLI preflight output | medium | mitigate | CLI prints structured issue codes and target id so failed gates can be recorded in verification artifacts. |
| T-11-10 | Information Disclosure | CLI output and live-check errors | high | mitigate | Output includes key names/resource names only and never prints token or secret values. |
| T-11-11 | Elevation of Privilege | Wrangler command execution | high | mitigate | Live checks pass argv arrays to an injected executor and include read-only checks only. |
| T-11-SC | Tampering | npm package metadata | high | mitigate | Reuse existing `tsx` lockfile version only; do not add package families or version upgrades in this plan. |
</threat_model>
