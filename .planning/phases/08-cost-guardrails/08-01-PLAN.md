<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| dashboard -> `/api/upload` | 管理端上传一旦没有明确 purpose，就会把任意资源重新带回 generic `images/` 成本黑洞 |
| `/api/upload` -> R2 + `media` | 错误 prefix 会同时污染存储成本与 `media` 元数据，后续 cleanup 难度显著上升 |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-08-01 | Tampering | `/api/upload` key generation | high | mitigate | 固定 purpose vocabulary 和 prefix map，彻底移除 generic `images/` key 生成 |
| T-08-02 | Elevation / Policy bypass | dashboard comic cover upload | high | mitigate | 把 `Comics.vue` 从 presign 侧门收回到 purpose-based upload helper |
| T-08-03 | Repudiation | manual upload provenance | medium | mitigate | 在 route tests 中锁定 purpose -> prefix mapping，避免后续“改回 images/ 但没人注意到” |
</threat_model>
