**新建 apps/movie-app/src/composables/useAuthGuard.ts：**

参考 PATTERNS.md line 463-487 的完整模式：

```typescript
import { useUserStore } from '../stores/user'

/**
 * 登录门控 composable。
 * requireLogin() 返回 false 时已执行跳转，调用方应立即 return。
 * 设计为下期可复用：nextPath 参数支持 Phase 4 历史/进度场景传入自定义路径。
 */
export function useAuthGuard() {
  const userStore = useUserStore()

  /**
   * 检查登录态，未登录则跳转到登录页并返回 false。
   * @param nextPath 登录后回弹路径，默认为当前页
   */
  function requireLogin(nextPath?: string): boolean {
    if (userStore.user) return true
    const target = nextPath ?? (window.location.pathname + window.location.search)
    window.location.href = `/auth/login?next=${encodeURIComponent(target)}`
    return false
  }

  return { requireLogin }
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → /auth/login | next 参数来自用户输入（URL query string），需要同源校验 |
| Vue composable → window.location | requireLogin 执行页面跳转，跳转目标由 next 参数决定 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03 | Spoofing | Open Redirect via next 参数 | mitigate | `login.vue` 的 `redirectPath` computed 使用 `new URL(raw, window.location.origin).origin === window.location.origin` 同源校验；跨域 URL 回退到 `/`（D-14） |
| T-02-14 | Spoofing | useAuthGuard 构造的 next 参数 | accept | `requireLogin` 中 next 参数由 `window.location.pathname + window.location.search` 构造，不接受外部输入；同源校验在 login.vue 侧兜底 |
| T-02-15 | Elevation of Privilege | 绕过 requireLogin 直接调用 API | accept | 组件层拦截是 UX 层防护，API 层有 `requireAuth` 作为真正的安全边界；组件层拦截只是避免 401 闪烁 |

</threat_model>
