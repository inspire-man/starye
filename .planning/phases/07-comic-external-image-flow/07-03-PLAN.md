<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Reader UI -> external image hosts | 图片失败是浏览器直接暴露出来的运行时现象，前端必须明确呈现和隔离它 |
| Reader progress -> public progress API | 错误的 completed 判定会把失败章节写成“已读完”，污染用户进度数据 |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-07-07 | Repudiation | Reader failure UI | medium | mitigate | 为每个失败页提供显式卡片和章节级汇总，防止“加载过/没加载过”无法判断 |
| T-07-08 | Tampering | progress persistence | high | mitigate | completed 判定必须要求至少一页成功加载，禁止空数组或全失败章节写成已完成 |
| T-07-09 | Spoofing / UX Integrity | open-original action | medium | mitigate | 外链打开统一使用 `noopener,noreferrer`，只展示页码和 host，不信任远端标题 |
</threat_model>
