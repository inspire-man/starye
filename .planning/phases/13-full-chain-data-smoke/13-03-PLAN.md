## Trust Boundaries

| Boundary | Description |
|---|---|
| Local service/env readiness -> smoke runner | Missing keys, schema, or services must stop before mutation. |
| Gateway -> Dashboard/public UI | Gateway routing and authenticated session are the contract; direct ports/API responses cannot impersonate UI proof. |
| D1/API/UI observations -> evidence tuple | A stale cache or different item can produce a superficially plausible but false success. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-13-10 | Tampering | local readiness | high | mitigate | Validate projection, schema, and services before one fixture call; no implicit repair beyond managed blocks/migrations. |
| T-13-11 | Spoofing | Dashboard proof | high | mitigate | Require Gateway-visible real session; redirects/roles become checkpoints, never mocked pass. |
| T-13-12 | Repudiation | cross-surface tuple | high | mitigate | D1/API/admin/viewer records must share target/run/code/id and bounded retry metadata. |
| T-13-13 | Information Disclosure | local artifacts | high | mitigate | Typed evidence writer excludes env values, cookies, headers, direct service origins, and prepared context. |
| T-13-14 | Denial of Service | local runner | medium | mitigate | Fixed one-item operation and bounded retry rather than open-ended crawl/poll behavior. |
