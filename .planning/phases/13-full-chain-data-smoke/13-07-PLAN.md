<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| Runner/browser adapter -> evidence writer | Observed external facts must be reduced to a redacted, typed receipt. |
| CLI/operator input -> surface observation | Input may select only the known pending tuple, never assert external success. |
| Provider/browser evidence -> terminal verifier | Structural files are untrusted until tuple-bound receipt coverage validates. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-13-28 | Repudiation | terminal smoke claim | high | mitigate | Require per-surface typed receipts derived from runner/provider/browser execution and validate them at terminal verification. |
| T-13-29 | Spoofing | surface CLI | high | mitigate | Remove caller-supplied success assertion; observe fixed routes with a repository-owned adapter and tuple checks. |
| T-13-30 | Information Disclosure | browser/provider capture | critical | mitigate | Allowlist only tuple/path/result/integrity metadata; forbid cookies, headers, raw endpoints, tokens, bodies, and prepared context. |
| T-13-31 | Tampering | evidence artifact | high | mitigate | Enforce deterministic renderer parity, exact mode/target/run pair selection, integrity binding, and receipt-to-row matching. |
| T-13-32 | Elevation of Privilege | remote observer | high | mitigate | Resolve selected canonical base from target profile and keep remote preflight/closed entry boundaries unchanged. |
</threat_model>
