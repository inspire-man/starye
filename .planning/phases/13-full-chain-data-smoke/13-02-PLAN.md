<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| Remote process environment -> prepared child | Credentials are present only in the authorized execution environment and must be allowlisted without serialization. |
| Prepared context -> crawler/D1 child | Context controls selected identity but must not become a generic operation, endpoint, or SQL channel. |
| Fixture payload -> service-auth sync route | A write request crosses into API/D1 and must remain one non-R18 upsert. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-13-05 | Spoofing | remote target selection | critical | mitigate | Explicit target plus smoke preflight, account match, and read-only ownership checks before any child. |
| T-13-06 | Elevation of Privilege | prepared execution | critical | mitigate | Closed registry, fixed argv, allowlisted environment, and no caller options or ambient identity. |
| T-13-07 | Tampering | fixture ingest | high | mitigate | Deterministic one-item non-R18 payload, service-auth transport, and no full crawler route. |
| T-13-08 | Information Disclosure | child output | critical | mitigate | Typed output allowlist and tests rejecting secret/context/endpoint leakage. |
| T-13-09 | Denial of Service | crawler scope | medium | mitigate | One-item cap with no retry loop or corpus traversal. |
</threat_model>
