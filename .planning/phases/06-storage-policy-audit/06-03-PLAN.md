<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| local audit script -> Cloudflare R2/D1 | 只允许 read-only inventory，不允许任何远端状态变更 |
| audit outputs -> future cleanup phases | 后续 phase 会据此判断删除风险与成本风险 |
| repo changes -> git commit | 本 phase 只允许文档、只读脚本、报告契约和验证工单进入提交 |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-06-07 | Tampering | `audit-r2-storage.ts` | high | mitigate | 以 D-14 为硬边界，脚本只实现 list/query/report 输出；验证工单检查无 destructive storage API 出现 |
| T-06-08 | Repudiation | `06-R2-AUDIT-DRY-RUN.md` + details files | medium | mitigate | 以 D-11、D-12、D-13 固定 Markdown/JSON/CSV 契约和字段，避免后续 phase 无法比较不同 dry-run 结果 |
| T-06-09 | Information Disclosure | script output | medium | mitigate | 输出只记录 prefix、sample key、count、size、DB reference status 与风险，不打印 access key、secret、token |
| T-06-10 | Elevation of Privilege | commit scope | medium | mitigate | `06-VERIFICATION.md` 强制在提交前运行 `npx gitnexus detect-changes --repo starye --scope all`，确认 phase 只影响预期文档与只读脚本 |
</threat_model>
