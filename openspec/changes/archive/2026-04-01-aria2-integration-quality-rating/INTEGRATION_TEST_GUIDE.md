# Aria2 集成和评分系统 - 集成测试指南

## 概述

本文档提供 Aria2 集成和评分系统的集成测试指南，包括环境准备、测试步骤和验证清单。

## 前置条件

### 1. 开发环境

- Node.js >= 20
- pnpm >= 8
- 本地数据库已迁移

### 2. Aria2 服务（可选，用于 Aria2 功能测试）

```bash
# macOS/Linux
aria2c --enable-rpc --rpc-listen-all --rpc-allow-origin-all

# Windows
aria2c.exe --enable-rpc --rpc-listen-all --rpc-allow-origin-all
```

默认 RPC 地址：`http://localhost:6800/jsonrpc`

### 3. 启动开发服务

```bash
# 终端 1: 启动 API 服务
pnpm --filter api run dev

# 终端 2: 启动前端服务
pnpm --filter movie-app run dev
```

### 第 2 部分: Aria2 集成测试 (任务 25.1-25.8)

#### 25.1 启动 Aria2 服务

```bash
aria2c --enable-rpc --rpc-listen-all --rpc-allow-origin-all
```

**验证：** Aria2 RPC 端点可访问

#### 25.2 配置 Aria2 连接

1. 登录前端应用
2. 访问 Aria2 设置页面（个人中心 → Aria2 设置）
3. 填写配置：
   - RPC URL: `http://localhost:6800/jsonrpc`
   - Secret: (留空或填写 Aria2 secret)
4. 点击"测试连接"

**预期结果：** 
- ✓ 连接成功提示
- ✓ 显示 Aria2 版本信息
- ✓ 连接状态指示器变为绿色

#### 25.3 测试添加单个磁链

1. 访问任意电影详情页
2. 找到一个磁力链接播放源
3. 点击"下载"按钮
4. 在弹窗中选择"添加到 Aria2"

**预期结果：**
- ✓ 任务添加成功提示
- ✓ 任务出现在 Aria2 任务面板
- ✓ 任务状态为"下载中"

**测试磁链示例：**
```
magnet:?xt=urn:btih:C9E85FDFE82E1D7E27F2E78B0F9C36F9E85D8C9A&dn=test
```

#### 25.4 验证任务在 Aria2 中可见

1. 打开 Aria2 任务面板
2. 确认任务在列表中
3. 检查任务详情（文件名、大小、状态）

**预期结果：**
- ✓ 任务列表显示新添加的任务
- ✓ 任务状态正确（active/waiting/paused）
- ✓ 进度条显示当前进度

#### 25.5 测试任务控制

**暂停任务：**
1. 点击任务的"暂停"按钮
2. 确认任务状态变为"已暂停"

**恢复任务：**
1. 点击任务的"恢复"按钮
2. 确认任务状态变为"下载中"

**删除任务：**
1. 点击任务的"删除"按钮
2. 确认任务从列表中移除

**预期结果：**
- ✓ 所有控制操作响应正常
- ✓ 状态更新实时反映在 UI
- ✓ 操作后 Aria2 实际状态与 UI 一致

#### 25.6 测试 WebSocket 实时进度

1. 添加一个新的下载任务
2. 观察进度条更新（至少 1 分钟）
3. 检查下载速度和 ETA 显示

**预期结果：**
- ✓ 进度条实时更新（每秒或每几秒）
- ✓ 下载速度显示正确（KB/s, MB/s）
- ✓ ETA 计算合理
- ✓ WebSocket 连接状态指示器为"已连接"

#### 25.7 测试降级逻辑

1. 在浏览器开发者工具中断开 WebSocket（Network → WS → 右键 → Block）
2. 或在 Aria2 设置中禁用 WebSocket
3. 观察系统行为

**预期结果：**
- ✓ 系统自动切换到 HTTP 轮询
- ✓ 显示降级通知（Toast）
- ✓ 任务进度仍然更新（但频率较低）
- ✓ 重新连接后自动恢复 WebSocket

#### 25.8 测试批量添加任务

1. 准备 10 个测试磁链
2. 在电影详情页批量选择播放源
3. 点击"批量下载"

**预期结果：**
- ✓ 所有任务成功添加
- ✓ 任务列表显示所有任务
- ✓ 批量操作反馈清晰（成功 X 个，失败 Y 个）

## 附录

### 常见问题

**Q: Aria2 连接失败怎么办？**
A: 检查 Aria2 服务是否运行，RPC 地址是否正确，防火墙是否阻止。

**Q: WebSocket 无法连接？**
A: 检查浏览器是否支持 WebSocket，Aria2 是否启用 WebSocket 支持。

**Q: 评分提交失败？**
A: 检查用户是否登录，评分范围是否正确（1-5 星），是否触发频率限制。
