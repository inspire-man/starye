## 为什么不使用 E2E 测试工具？

虽然可以使用 Playwright/Cypress 实现 UI 自动化测试，但在当前阶段**不推荐**，原因：

### 缺点

1. **配置复杂**
   - 需要设置测试环境
   - 需要 mock 用户认证
   - 需要 mock Aria2 服务

2. **维护成本高**
   - 测试脚本容易随 UI 变化而失效
   - 需要持续更新选择器和断言
   - 调试困难

3. **执行时间长**
   - E2E 测试通常需要 10-30 分钟
   - 比手动测试慢

4. **覆盖率有限**
   - 无法测试用户体验
   - 无法验证视觉效果
   - 无法测试网络波动等真实场景

### 优点（未来可考虑）

- 回归测试自动化
- CI/CD 集成
- 减少重复劳动

### 建议

**当前阶段**: 手动测试  
**稳定版本后**: 考虑添加 E2E 测试（关键路径）
