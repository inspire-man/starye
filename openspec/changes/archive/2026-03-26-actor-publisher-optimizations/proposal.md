# 女优/厂商功能后续优化

## 背景

在 `actor-publisher-relations` 变更中，我们完成了女优/厂商关系的核心功能。本变更聚焦于性能优化、用户体验增强和运维改进。

## 目标

1. **增强后台管理界面**：提升女优/厂商数据管理效率
2. **提升热门女优/厂商列表的访问性能**
3. **增强女优数据管理能力**（别名、关系图谱）
4. **改善用户体验**（收藏功能）
5. **优化爬虫稳定性和数据清理**

## 范围

### 包含

- **后台管理 UI 增强**（新增）
  - 女优/厂商表格新增列：爬取状态、国籍/国家、创建时间
  - 优化筛选器：改进爬取状态筛选、添加国籍筛选
  - 统一空值显示、时间格式
- 为热门女优/厂商列表添加 Cloudflare KV 缓存
- 实现女优别名管理界面
- 添加女优合作关系图谱
- 支持用户收藏女优/厂商
- 为爬虫添加代理池
- 定期清理备份字段

### 不包含

- 不涉及核心功能的修改（已在 actor-publisher-relations 中完成）
- 不涉及数据库 schema 变更（字段已存在）

## 优先级

- **P0: 后台管理 UI 增强** - 直接提升日常管理效率 MUST
- P0: 缓存优化 - 直接影响性能 MUST
- P1: 数据清理 - 减少数据库负担 SHALL
- P2: 代理池 - 提升爬虫稳定性 SHALL
- P3: 用户功能（别名、图谱、收藏）- 增强体验 MAY

## 交付物

- Cloudflare KV 缓存实现
- Dashboard 女优别名管理页面
- 女优关系图谱组件
- 用户收藏功能
- 爬虫代理池
- 数据清理脚本

## 成功标准

- 热门列表响应时间 < 100ms
- 别名管理界面可用
- 关系图谱展示正常
- 用户可以收藏并查看收藏列表
- 爬虫代理池自动切换
- 备份字段清理脚本可执行
