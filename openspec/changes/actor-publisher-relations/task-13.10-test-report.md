## 手动测试指南

由于 Dashboard 使用 Better Auth 认证系统，需要通过浏览器手动测试。

### 测试步骤

#### 1. 启动开发环境

```bash
# 终端 1: 启动 API
cd apps/api && pnpm dev

# 终端 2: 启动 Dashboard
cd apps/dashboard && pnpm dev
```

#### 2. 登录 Dashboard

1. 打开浏览器访问 Dashboard (通常是 http://localhost:5173)
2. 使用 GitHub OAuth 或其他方式登录
3. 确保账号有 admin 或 movie_admin 权限

#### 3. 测试女优选择器

**测试用例 3.1: 搜索功能**
1. 进入"电影管理"页面
2. 点击任意一部电影，打开编辑弹窗
3. 在"女优"选择器的搜索框中输入女优名称（例如："明"）
4. **预期结果**:
   - 搜索结果实时显示
   - 显示匹配的女优列表，包含名称和作品数量
   - 已选择的女优有高亮标记

**测试用例 3.2: 多选功能**
1. 在搜索结果中点击或勾选一个女优
2. 再选择另一个女优
3. **预期结果**:
   - 女优添加到"已选择"列表
   - 显示"已选择 (N)"标题
   - 每个项显示名称和排序序号

**测试用例 3.3: 拖拽排序**
1. 在已选择列表中，拖动左侧的"☰"图标
2. 移动到不同位置释放
3. **预期结果**:
   - 拖拽过程中光标变为 grabbing
   - 释放后项目移动到新位置
   - 右侧的排序序号（#1, #2, ...）自动更新

**测试用例 3.4: 删除功能**
1. 点击已选择项右侧的"×"按钮
2. **预期结果**:
   - 项目立即从列表移除
   - 其他项目的排序序号自动调整
   - 总数"已选择 (N)"正确更新

**测试用例 3.5: 快速创建**
1. 在搜索框中输入一个不存在的女优名称（例如："测试女优123"）
2. 等待搜索完成，无结果
3. 点击"+ 创建"按钮
4. 在弹出的创建表单中确认名称
5. 点击"创建"按钮
6. **预期结果**:
   - 搜索无结果时显示"+ 创建"按钮
   - 点击后弹出创建表单（模态窗口）
   - 表单自动填充搜索内容
   - 创建成功后自动添加到已选择列表
   - 创建表单自动关闭
   - 搜索框清空
